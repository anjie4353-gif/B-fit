# B-Fit Brand

## Official assets

| Asset | Path |
|-------|------|
| Logo mark (transparent) | `public/brand/logo-mark.png` |
| Wordmark (CSS, preferred) | `components/brand/brand-wordmark.tsx` |
| Lockup (mark + name) | `components/brand/brand-lockup.tsx` |

Source files: user provides `image copy.png` (mark) and `image copy 2.png` (name reference).

## Rules

1. **No white boxes** — remove white with ImageMagick `-fuzz 14% -transparent white -trim`
2. **Wordmark = CSS text**, not pasted PNG (premium, scales cleanly)
3. **App background blend** — PWA icons use `#f5f3ff` (same as `--background`)
4. **Colors** — `lib/brand.ts`: `BRAND_NAVY`, `BRAND_TEAL`, tagline gray

## Home screen icon (WhatsApp-style)

After PWA install, phone home screen must show the B-Fit logo mark.

Required assets (generated by `npm run brand:assets`):
- `icon-192.png` / `icon-512.png` — exact square sizes, logo centered on `#f5f3ff`
- `maskable-icon-192.png` / `maskable-icon-512.png` — Android adaptive safe zone
- `apple-touch-icon.png` — 180×180 for iOS Add to Home Screen
- `favicon.ico` — browser tab
- Android `mipmap-*/ic_launcher*.png` — native APK

`app/manifest.ts` must list both `any` and `maskable` purposes.

## Icon updates for installed users (v1.3.3+)

Phones often freeze the home-screen icon at install time. To update old users:

1. **Versioned URLs** — `lib/pwa/icon-urls.ts` appends `?v=APP_VERSION` to manifest + icon `src`
2. **No icon precache** — `public/sw.js` must NOT precache `/icons/*` or manifest; use network-first
3. **Bump on deploy** — `APP_VERSION` in `lib/pwa/version.ts` + `CACHE` in `public/sw.js`
4. **no-cache headers** — `next.config.ts` for `/icons/*`, `/manifest.webmanifest`, `/sw.js`
5. **Prompt** — `IconRefreshPrompt` banner for standalone users where `needsIconRefresh()` is true
6. **Help page** — `/install#icon-refresh` with new icon preview + re-add steps; `markIconVersionSynced()` on confirm

**Android:** may auto-update WebAPK icon after app update + versioned manifest URLs.  
**iPhone:** user must delete old icon → Safari → Share → Add to Home Screen.

Never call `markIconVersionSynced()` on standalone mount alone — only on fresh install accept or user confirms refresh.
